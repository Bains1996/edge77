## 3. Technical Integration Requirements Checklist

### Pre-Integration Checklist

- [ ] **API documentation obtained** from TMS partner (REST/SOAP/GraphQL)
- [ ] **Authentication method confirmed** (API key, OAuth 2.0, basic auth)
- [ ] **Rate limit** documented (requests/min, daily cap)
- [ ] **Sandbox/test environment** access granted
- [ ] **Production environment** credentials (separate from sandbox)
- [ ] **Data retention policy** agreed with partner (how long EDGE77 stores data)
- [ ] **SLA for uptime** documented (partner API availability requirement)
- [ ] **Webhook support** confirmed (for real-time invoice/status updates)

### Data Exchange Specifications

#### What EDGE77 Receives (from TMS)

| Data Object | Format | Required | Notes |
|---|---|---|---|
| Invoice data | JSON/XML/EDI 210 | Yes | Line items, charges, reference numbers |
| Carrier contracts | CSV/JSON | Yes | Rate agreements, accessorial schedules |
| Shipment data | JSON/XML | Recommended | For context on invoice validation |
| Carrier master data | JSON | Recommended | Name, MC#, address, contact |

#### What EDGE77 Sends (back to TMS)

| Data Object | Format | Trigger | Notes |
|---|---|---|---|
| Audit results | JSON | Per invoice processed | Pass/fail status, discrepancy details |
| Dispute documents | PDF | Manual or auto | Attached to audit result |
| Recovery summary | JSON/CSV | Daily batch | Aggregated recovery data |
| Carrier error report | JSON/CSV | Weekly | Trend analysis |

### Integration Maturity Levels

| Level | Description | Implementation | Timeline |
|---|---|---|---|
| **1 — File-Based** | Export/import via CSV or flat file | No API needed | 1-2 weeks |
| **2 — API Sync** | REST API integration for real-time sync | Basic API connector | 4-6 weeks |
| **3 — Embedded** | EDGE77 UI embedded as iframe/widget in TMS | OAuth + iframe | 8-12 weeks |
| **4 — Deep Native** | Full bidirectional integration with data transformation | Enterprise | 12-16 weeks |

### Security Requirements

- [ ] SOC 2 Type II compliance documentation shared with partner
- [ ] IP allowlist for API access
- [ ] Data encryption standard confirmed (minimum: TLS 1.3 in transit, AES-256 at rest)
- [ ] Data deletion/retention policy documented
- [ ] Incident response plan shared
- [ ] Penetration test results (annual)
- [ ] Sub-processor list (if any)
